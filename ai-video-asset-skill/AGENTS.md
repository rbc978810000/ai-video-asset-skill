# 项目协作记忆

## 中文工程产物命名规则

本项目所有用户可见的生成工程目录、工程文件、调研产物、分镜产物、生图产物、审核产物、导出产物，默认必须使用中文命名。

必须遵守：

- 每个素材项目只使用一个工程目录；`mine-market`、`run`、参考帧采集、人工审核、导出都写入同一个项目目录，不因英文内部任务名另起目录。
- 工程根目录和产物目录使用中文，例如 `00_输入`、`00_调研`、`01_分镜`、`02_生图提示词`、`03_图片`、`04_扩展角度`、`05_审核`、`06_导出`。
- 关键工程文件使用中文，例如 `项目清单.json`、`选题简报.md`、`风格设定.json`、`分镜总表.json`、`镜头_001.json`、`镜头_001_生图提示词.json`、`当前图.png`、`审核记录.json`、`交付索引.csv`。
- 调研子目录使用中文，例如 `市场反挖`、`参考图`、`视频参考帧`、`精选参考帧`、`来源`、`镜头`、`帧图片`、`缩略图`、`分析拼图`。
- 图片版本文件使用中文序号，例如 `版本1.png`、`版本2.png`、`版本3.png`；当前采用版本复制为 `当前图.png`。
- 文档、测试、Electron 查看器和脚本中的路径示例必须与上述中文产物契约保持一致。

允许保留英文：

- Python 模块名、函数名、变量名、CLI 参数名、JSON schema 名称、JSON 字段键、第三方 API 字段和内部 ID 可以保留英文。
- 内部数据 ID 如 `shot_id=shot_001`、`frame_id`、`source_id` 可以保留英文，但写入磁盘的目录或文件名必须映射为中文可读形式，例如 `shot_001` 对应 `镜头_001`。

改动前后检查：

- 修改任何会写入文件系统的逻辑后，必须搜索旧英文产物名，确认没有新增 `00_inputs`、`00_research`、`project_manifest.json`、`current.png`、`review.json`、`market_mining/`、`video_reference_assets/`、`selected_reference_frames/`、`reference_assets/` 等旧路径。
- 相关测试也必须使用中文产物路径，不能把旧英文目录作为期望结果。
- 如需迁移历史 `output/` 旧工程，必须先单独说明迁移范围并得到用户确认；普通代码修改不得擅自重命名或删除历史产物。
